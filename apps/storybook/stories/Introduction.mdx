# Afenda Storybook

Workshop for `@repo/design-system` — Afenda UI primitives, composed Blocks, Foundations tokens, and Governance readiness evidence.

## Layers

| Sidebar | Purpose |
| --- | --- |
| **Afenda UI** | Product primitives (`stories/afenda-ui/`) |
| **Blocks** | Composed operator workflows (`stories/blocks/`) |
| **Foundations** | Token reference and usage policy |
| **Governance** | Internal readiness matrices (hidden by default via `internal` tag) |

Internal shadcn source lives in `packages/design-system/components/ui` and is **not** published as Storybook stories.

## Toolbars

- **Theme** — light / dark via `@storybook/addon-themes`
- **Layout** — center, padded, fullscreen, surface, frame
- **Density** — comfortable / compact
- **Viewport** — mobile through dense dashboard

## Commands

```bash
pnpm --filter storybook dev
pnpm --filter storybook check          # typecheck + hygiene + static build
pnpm --filter storybook hygiene        # story structure and import gates
pnpm --filter storybook test-storybook:interaction
```

See `.storybook/ECOSYSTEM.md` and `.storybook/TESTING.md` for MCP, Vite, and test lanes.

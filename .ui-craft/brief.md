## 1. Product purpose

Afenda is a governed operator workspace for tenant-scoped SaaS administration, audit evidence, and domain workflows.

## 2. Primary user

Operations and compliance admins working repeatedly in dense dashboards on laptop screens, where fast scanning and correct access boundaries matter more than visual spectacle.

## 3. Principles

1. Correctness beats speed.
2. Show the data, not the decoration.
3. Density is a feature when hierarchy stays legible.
4. Color earns its place through status, focus, or wayfinding.
5. Code tokens in `packages/design-system/tokens/tokens.json` remain the source of truth.
6. Quiet interfaces, loud decisions.

## 4. Success metric for the surface

Users can identify the current tenant, risk state, and next required operational action within 30 seconds of landing on an authenticated dashboard surface.

## 5. Out of scope

- Does not use marketing-page hero composition inside authenticated app screens.
- Does not use multiple decorative accent colors in one operational viewport.
- Does not promote page-specific dashboard blocks into `@repo/design-system` before reuse is proven.
- Does not add high-motion choreography to repeated operator workflows.
- Does not use warmth tokens in tables, forms, dense dashboards, admin CRUD, audit logs, or approval queues.
